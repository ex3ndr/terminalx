Why We Chose a Classless Approach for TerminalX CSS

Mar 25, 2026 8 min read by @terminalx

When we started building TerminalX CSS, we had one clear goal: make semantic HTML look good without adding a single class to your markup. No .container, no .text-lg, no .flex-col. Just HTML elements, styled beautifully out of the box.

This post explains why we took that path, what we learned, and where we think classless CSS fits in the modern toolchain.

The Problem with Class-First CSS

Utility-first frameworks like Tailwind are incredible for building complex UIs. But for content-heavy pages — documentation, blogs, changelogs, settings panels — they add friction. You end up decorating every element:

<!-- Tailwind approach -->
<h1 class="text-2xl font-bold text-gray-100 mb-4">Title</h1>
<p class="text-gray-400 leading-relaxed mb-6">Content...</p>
<a class="text-orange-500 hover:text-orange-400 underline">Link</a>

<!-- TerminalX approach -->
<h1>Title</h1>
<p>Content...</p>
<a href="#">Link</a>

For rendered markdown, CMS output, or any content you don't fully control, classless styling is the only practical option1.

Design Decisions

Monospace Body, Sans-Serif Headings

Most CSS frameworks default to sans-serif everywhere. We flipped this: Geist Mono for body text gives the terminal aesthetic, while Geist (sans) for headings creates a clear visual hierarchy and improves scannability.

Typography is what language looks like. Using monospace as the default isn't a limitation — it's a statement about the kind of interfaces we're building.

Internal design review notes

Soft Dark, Not Harsh Dark

Pure black (#000) backgrounds with pure white (#fff) text create harsh contrast that strains the eyes. We went with #0c0c0c for the background and #e8e8e8 for text — subtle but meaningfully softer. The Factory.ai aesthetic was our north star here.

The color hierarchy follows a clear pattern:

Layer Color Use
Background #0c0c0c Page base
Surface #141414 Cards, code blocks, inputs
Elevated #1a1a1a Buttons, kbd, raised elements
Hover #222222 Interactive states

CSS Variables All the Way Down

Every color, spacing value, and font stack is a CSS custom property. No build tools, no Sass, no PostCSS. Override --accent and the entire theme shifts — links, focus rings, progress bars, alerts, everything.

:root {
  /* Swap orange for blue with one line */
  --accent: #3b82f6;
  --accent-hover: #60a5fa;
  --accent-muted: rgba(59, 130, 246, 0.15);
}

What Classless Can't Do

We're not pretending this replaces Tailwind or CSS modules. Classless CSS is intentionally limited:

  • Layout — You still need classes for grids, flexbox layouts, and responsive patterns
  • Complex components — Modals, dropdowns, tabs need JS and custom markup
  • Fine-grained control — Per-element overrides need custom CSS or utility classes

TerminalX addresses this by providing a small set of optional utility classes (.card, .alert, .btn-ghost) for patterns that can't be expressed through HTML semantics alone. But the core promise holds: if you write clean HTML, it looks good by default.

Try It

Drop a single <link> tag into your HTML and start writing. No npm install, no config file, no build step2.

<link rel="stylesheet"
  href="https://unpkg.com/terminalx/terminalx.css">

That's it. Your semantic HTML is now styled.
